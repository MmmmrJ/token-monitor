# Token Monitor v1.1.3 实施计划

> 当前基线：`v1.1.2`
>
> 目标版本：`v1.1.3`
>
> 计划日期：2026-07-20
>
> 主要实施平台：macOS；Windows 通过 GitHub Actions 与发布产物完成交叉验证

## 1. 当前进度

`main` 当前版本为 `v1.1.2`，远端 Tag 与 Release 已发布。现有发布流水线能够构建 Windows x64 MSI/NSIS 和 macOS Universal DMG。

v1.1.2 已完成以下基础能力：

- Codex/Cursor Provider 数据映射、状态模型与缓存隔离。
- Provider 快速切换时的请求竞态防护。
- 中英文状态、部分可用、离线缓存和明确错误原因。
- 前端测试、Rust 测试、Clippy、版本一致性检查及双平台 CI。
- `100% × 100%` 贴边画布、八向缩放、`480 × 300` 最小尺寸、28px 连续圆角和透明边缘。

v1.1.3 不重新设计主界面。贴边画布与缩放作为回归项保留，本版本的实际开发重点是关闭到托盘、单实例和 autostart 迁移。

## 2. 版本目标与边界

v1.1.3 需要让 Token Monitor 在 Windows 和 macOS 上具备稳定、可预测的桌面应用生命周期：

1. 关闭主窗口时隐藏到托盘，只有明确的退出操作才结束进程。
2. 重复启动时复用现有进程并恢复主窗口。
3. 将旧的 `Codex Usage Monitor` 登录启动项安全迁移为 `Token Monitor`。
4. 以前端显示操作系统的真实 autostart 状态，并提供可恢复的内联错误反馈。

本版本不包含后台刷新协调器、额度阈值提醒、应用内 Updater、新 Provider、历史数据或 Dashboard；这些能力继续保留在 v1.2.0 以后。

## 3. 原生窗口生命周期

### 3.1 统一窗口恢复入口

在 Rust 侧新增 `show_main_window` helper，并固定按以下顺序恢复主窗口：

```text
show → unminimize → set_focus
```

托盘“打开”、托盘图标双击和第二实例回调必须共用这个 helper，避免不同入口出现行为差异。单步失败不应 panic；应继续尝试后续步骤，使窗口尽可能恢复。

### 3.2 关闭到托盘

增加进程级 `LifecycleState`，使用原子布尔值记录应用是否正在执行真正退出。

- `main` 窗口收到 `CloseRequested` 且未处于退出流程时，调用 `prevent_close()`，随后隐藏窗口。
- Windows 关闭按钮、Alt+F4、macOS Cmd+W 均执行隐藏，不销毁 WebView 和运行时状态。
- 托盘“退出”先设置退出标志，再调用 `app.exit(0)`。
- macOS Cmd+Q 进入 `ExitRequested` 时设置退出标志，并允许 Tauri 正常结束进程。
- 不在 `ExitRequested` 中调用 `prevent_exit()`。

### 3.3 单实例

在 `src-tauri/Cargo.toml` 中加入：

```toml
tauri-plugin-single-instance = "2"
```

将 Single Instance 注册为 `tauri::Builder` 的第一个插件。第二实例回调只调用 `show_main_window`，忽略传入的命令行参数与工作目录，不启动新的托盘、刷新器或窗口状态。

实现应遵循 Tauri v2 官方 Single Instance 插件约定：

- <https://v2.tauri.app/zh-cn/plugin/single-instance/>

## 4. Autostart 迁移与状态同步

### 4.1 注册名称与兼容边界

- 将 `tauri-plugin-autostart` 的 `app_name` 从 `Codex Usage Monitor` 改为 `Token Monitor`。
- 保留 `com.codexusagemonitor.desktop` identifier、Cargo package 名、内部二进制名和现有 localStorage key。
- 显式加入 `auto-launch = "0.5"`，仅用于检测和清理旧名称启动项；主启动项仍由 `tauri-plugin-autostart` 管理。
- 不新增 Store 插件或迁移配置文件。旧启动项存在即表示迁移尚未完成，旧项成功删除即表示迁移完成。

平台目标：

- Windows：旧名称位于当前用户的 `Run`/`StartupApproved` 启动注册项。
- macOS：旧名称为 `~/Library/LaunchAgents/Codex Usage Monitor.plist`。

### 4.2 公开命令和返回结构

新增或调整为以下两个 Rust command：

```text
initialize_start_at_login(preferredEnabled) -> AutostartStatus
set_start_at_login(enabled) -> AutostartStatus
```

返回结构使用 camelCase：

```text
AutostartStatus
  enabled: boolean
  migrated: boolean
  errorKind:
    null | query_failed | enable_failed |
    cleanup_failed | disable_failed
```

`enabled` 必须表示操作结束后系统中的有效状态；`migrated` 仅在本次调用成功删除旧项时为 `true`。WebView 只接收稳定错误类型，不接收底层错误文本、完整可执行路径或启动项内容。

### 4.3 初始化迁移算法

前端启动后调用 `initialize_start_at_login`，并把现有 `preferences.startAtLogin` 作为 `preferredEnabled` 传入。

迁移顺序固定如下：

1. 查询 `Token Monitor` 新启动项和旧启动项状态。
2. 计算目标状态：新项已启用、旧项已启用或保存偏好为启用时，目标均为启用。
3. 目标为启用但新项未启用时，创建新项并再次查询验证。
4. 只有新项验证成功后才删除旧项，并再次验证旧项已移除。
5. 新项创建失败时保留旧项，返回 `enable_failed`。
6. 新项已生效但旧项清理失败时返回 `cleanup_failed`；Single Instance 必须阻止重复启动产生第二个有效实例。
7. 重复调用迁移不得重复创建、报错或改变已经稳定的状态。

### 4.4 用户主动切换

- 启用时先创建并验证新项，再清理可能残留的旧项。
- 禁用时同时清理新旧项，并查询最终状态。
- 任一操作失败时，通过 `AutostartStatus` 返回实际状态和错误类型，不把前端开关停留在乐观状态。

### 4.5 前端交互

- 初始化和切换进行中禁用“登录时启动”控件。
- 命令完成后以返回的 `enabled` 同步 checkbox、`preferences.startAtLogin` 和 localStorage。
- 操作失败时回滚到返回的真实状态，在设置项内部显示中英文错误；成功后清除错误。
- 错误提示使用 `role="status"` 或等价的可访问状态区域，不使用 Toast。
- 普通浏览器和 `?preview=1` 模式不得调用或伪造原生 autostart 状态。

Tauri Autostart 官方参考：

- <https://v2.tauri.app/plugin/autostart/>

## 6. 自动测试

为 autostart 迁移抽取不依赖真实系统启动项的纯决策逻辑，至少覆盖：

- 新旧项均不存在、偏好关闭：保持关闭。
- 新旧项均不存在、偏好启用：创建新项。
- 仅旧项启用：创建并验证新项后删除旧项。
- 仅新项启用：保持启用，不执行多余操作。
- 新旧项均启用：保留新项并清理旧项。
- 新项创建失败：旧项不被删除。
- 旧项清理失败：返回 `cleanup_failed`，有效状态仍为启用。
- 禁用时新旧项均被清理。
- 同一迁移连续执行两次，第二次无副作用。

常规检查命令：

```bash
npm test
npm run check:versions
npm run build
cargo fmt --manifest-path src-tauri/Cargo.toml --check
cargo test --manifest-path src-tauri/Cargo.toml
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings
```

## 7. macOS 手工验收

### 生命周期

- Cmd+W 隐藏窗口，Dock/托盘入口可恢复窗口和焦点。
- 托盘“退出”和 Cmd+Q 真正结束进程。
- 主窗口最小化或隐藏后再次启动应用，旧窗口被恢复且聚焦。
- 连续启动应用 10 次，活动监视器中始终只有一个 Token Monitor 实例。

### Autostart

- 无启动项且设置关闭：启动后保持关闭。
- 开启设置后出现 `Token Monitor` LaunchAgent，重启应用后开关仍为开启。
- 预置旧 `Codex Usage Monitor` LaunchAgent 后启动新版：新项成功创建，旧项随后删除。
- 同时存在新旧项时只清理旧项，不影响新项。
- 模拟新项创建失败时旧项仍存在，设置界面显示内联错误。
- 关闭设置后新旧启动项均不存在，界面状态与系统一致。

### UI 回归

- 标题栏拖动和八向缩放正常，最小尺寸为 `480 × 300`。
- 四边无透明留白或白色角块，28px 圆角连续且无外投影。
- `.dual-view` / `.focus-view` 切换不刷新或改写 Snapshot。
- Codex/Cursor 快切、中文/English、始终置顶和托盘刷新均保持正常。

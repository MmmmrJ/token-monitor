# Token Monitor v1.2.0 实施计划

> 当前基线：`v1.1.3`
>
> 目标版本：`v1.2.0`
>
> 计划日期：2026-07-24
>
> 主要实施平台：macOS；Windows 通过 GitHub Actions 与发布产物完成交叉验证

## 1. 当前进度

`main` 当前版本为 `v1.1.3`，远端 Tag `v1.1.3` 已推送。现有发布流水线能构建 Windows x64 MSI/NSIS 和 macOS Universal DMG。

v1.1.3 已完成：关闭到托盘、单实例、autostart 迁移、贴边画布回归项。

v1.2.0 不重新设计主界面。开发重点是后台刷新协调器、托盘效率、额度提醒、设置同步与 Updater bootstrap。

## 2. 版本目标与边界

1. 窗口隐藏后仍按 60 秒周期刷新活动 Provider，并通过 `monitor:snapshot` 推送非敏感快照。
2. 托盘提供双语摘要、Provider/视图切换、刷新、置顶、显示与退出。
3. 可选额度提醒：默认总开关关闭、默认仅启用 10% 阈值；去重状态跨重启保留。
4. 引入可验签的 Updater 客户端与单一 `latest.json` 聚合发布流程。

本版本不包含「关于与诊断」完整页、平台公证/Authenticode 强制、第三个 Provider、历史 Dashboard。

## 3. MonitorCoordinator

### 3.1 行为

- 活动 Provider 默认每 60 秒刷新。
- 每个 Provider 最多一个进行中请求；手动刷新复用进行中的 Future。
- 连续网络/服务失败退避：60 → 120 → 300 秒，上限 5 分钟；成功后重置为 60 秒。
- 单调时钟相邻 tick 间隔超过 90 秒视为睡眠/唤醒，立即调度活动 Provider。
- 通知规则启用的非活动 Provider 也可后台刷新。
- 成功快照写入 Provider 独立内存缓存。
- 通过 `monitor:snapshot` 事件推给 WebView。

WebView 不再依赖页面 `setInterval` 维持后台监控；浏览器与 `?preview=1` 不得伪装原生协调器。

### 3.2 命令

保留 `refresh_monitor_data` / `get_monitor_snapshot`，内部委托协调器，确保手动刷新与后台单飞一致。

## 4. 托盘

菜单项：

- Codex / Cursor 摘要（百分比 + 最近重置倒计时）
- 切换 Provider
- 切换双环 / 聚焦
- 手动刷新
- 始终置顶
- 显示窗口 / 打开设置
- 退出

Windows 单击托盘显示窗口；macOS 保持菜单栏标准行为。文案跟随中文 / English。

## 5. 额度提醒与偏好

### 5.1 结构

```text
MonitoringPreferences
  enabled: boolean
  providerRules: Map<Provider, ProviderAlertRules>

ProviderAlertRules
  fiveHour / sevenDay: WindowAlertRule

WindowAlertRule
  enabled: boolean
  thresholdsRemaining: [20, 10, 5]  # 默认仅 10 启用
  notifyOnReset: boolean            # 默认 true
```

`AlertRuntimeState` 按 provider/window 保存 `cycleKey`、`notifiedThresholds`、`resetNotified`、`lastRemainingPercent`。

### 5.2 规则

- 总开关默认关；开启时才申请通知权限；拒绝则回滚并内联引导。
- 同一 Provider/窗口/周期/阈值只提醒一次；`resetsAt` 变化换周期。
- 只对 fresh、非 cached、真实存在的窗口触发。
- 无 `resetsAt`：回升 ≥5pt 才重新布防；不发 Reset 恢复提醒。
- 配置原子写入（临时文件 + rename）；损坏回退默认并保留坏文件。

### 5.3 命令

```text
get_monitor_preferences
set_monitor_preferences
sync_ui_preferences
```

设置面板分区：显示与窗口 / 额度提醒。失败回滚并显示中英内联错误，不用 Toast。

## 6. Updater bootstrap

### 6.1 本地密钥生成（一次性）

```bash
npm run tauri -- signer generate -w ~/.tauri/token-monitor.key
```

- 公钥写入 `src-tauri/tauri.conf.json` 的 `plugins.updater.pubkey`
- 私钥路径与密码只保存在本机与 GitHub Secrets，**永不提交仓库**
- Secrets 建议名称：
  - `TAURI_SIGNING_PRIVATE_KEY`
  - `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

未配置 Secrets 时，正式 Stable Tag 的 updater 签名门禁不得放行。

### 6.2 客户端

- `tauri-plugin-updater` + `tauri-plugin-process`
- `createUpdaterArtifacts: true`
- Stable endpoint：GitHub Releases `latest.json`
- 启动 15 秒后检查一次，此后每 24 小时
- 发现更新只显示顶部短条；下载 / 安装 / 重启需用户确认

### 6.3 Release

- 平台 job 只上传 Actions artifact
- 聚合 job 创建 Release、上传安装包与 updater 资产、生成唯一 `latest.json`
- 后置校验：DMG、MSI、NSIS EXE、平台 updater artifact、签名文件、`latest.json`

## 7. 实施顺序

### 阶段 A：协调器

1. 创建分支 `codex/v1.2.0-background-monitor`
2. 实现 `MonitorCoordinator` 与 `monitor:snapshot`
3. 前端订阅事件并移除 60s `setInterval` 后台刷新
4. Rust 单飞 / 退避 / tick gap 单元测试

### 阶段 B：托盘与提醒

1. 扩展托盘菜单与 Windows 单击行为
2. 偏好 / 运行时状态持久化与提醒决策纯函数
3. 设置分区 UI 与命令
4. 提醒去重测试

### 阶段 C：Updater 与发布

1. 本地生成密钥，公钥入库
2. 接入 updater/process 插件与前端检查流
3. 改造 `release.yml` 聚合与完整性校验
4. README / ROADMAP / 版本 `1.2.0`

## 9. 完成标准

- 窗口隐藏后托盘与通知继续刷新。
- 每个 Provider 同时最多一个上游请求。
- 同一阈值/周期只通知一次；权限拒绝回滚总开关。
- Release 含可验签 updater artifact 与唯一 `latest.json`（Secrets 就绪后）。
- 贴边画布、Provider 快切、autostart、关闭到托盘无回归。
